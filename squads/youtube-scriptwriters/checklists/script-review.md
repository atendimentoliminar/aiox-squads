# Script Review Checklist

Review checklist for YouTube scripts. Used by `george-blackman` (*review) and `joba` (*analise).

## How to Use
For each item, mark: PASS / FAIL / N/A
If FAIL, add a note explaining what needs fixing.

---

## 1. Hook Quality

| # | Check | Status | Notes |
|---|-------|--------|-------|
| 1.1 | Hook is in the first 5-30 seconds (no generic greetings before it) | | |
| 1.2 | Hook is SPECIFIC — names the audience, transformation, or stakes | | |
| 1.3 | Hook opens a curiosity gap that the video will close | | |
| 1.4 | First sentence creates immediate impact (surprise, identification, or curiosity) | | |

## 2. Structure & Flow

| # | Check | Status | Notes |
|---|-------|--------|-------|
| 2.1 | Grand Payoff / Promessa Principal is clearly identified | | |
| 2.2 | Segments are defined with clear transitions | | |
| 2.3 | Each segment has a mini payoff / mini promessa | | |
| 2.4 | Mini payoffs are spaced every 60-90 seconds | | |
| 2.5 | Curiosity gaps chain between segments (closing one opens next) | | |
| 2.6 | No unresolved curiosity gaps at the end | | |
| 2.7 | Grand Payoff is delivered near the end, not too early | | |
| 2.8 | Video ends shortly after Grand Payoff (no unnecessary summary) | | |

## 3. Content Quality

| # | Check | Status | Notes |
|---|-------|--------|-------|
| 3.1 | Language is specific — no vague terms ("important", "interesting", "incredible") | | |
| 3.2 | Every sentence earns its place (no filler) | | |
| 3.3 | Promises made in hook are fulfilled in the body | | |
| 3.4 | Information is progressively revealed (not all at once) | | |
| 3.5 | Tone is conversational, not corporate | | |

## 4. PCCE Analysis (George Blackman)

Label each sentence: **P**rogression / **C**onflict / **C**uriosity / **E**motion

| # | Check | Status | Notes |
|---|-------|--------|-------|
| 4.1 | Every sentence maps to at least one PCCE label | | |
| 4.2 | No sentences without a label (candidates for cutting) | | |
| 4.3 | Balance across P, C, C, E (not all Progression) | | |

## 5. Retention Optimization (Joba)

| # | Check | Status | Notes |
|---|-------|--------|-------|
| 5.1 | First 30 seconds use at least 1 Pilar de Conexao | | |
| 5.2 | Mini promessas every 60-90 seconds throughout | | |
| 5.3 | No section longer than 90s without a retention hook | | |
| 5.4 | Strongest mini promessa is in the middle (where retention usually drops) | | |
| 5.5 | No signals of ending before delivering the main payoff | | |

## 6. CTA / End Screen

| # | Check | Status | Notes |
|---|-------|--------|-------|
| 6.1 | Main payoff delivered BEFORE the CTA | | |
| 6.2 | CTA feels natural, not forced | | |
| 6.3 | Only ONE clear call to action | | |
| 6.4 | CTA connects to the value just delivered | | |

## 7. Read-Aloud Test

| # | Check | Status | Notes |
|---|-------|--------|-------|
| 7.1 | Script sounds natural when read aloud | | |
| 7.2 | No sentences that "trip the tongue" | | |
| 7.3 | Pacing feels right (not too dense, not too sparse) | | |

---

## Summary

| Category | Score | Threshold |
|----------|-------|-----------|
| Hook Quality | /4 | >= 3 |
| Structure & Flow | /8 | >= 6 |
| Content Quality | /5 | >= 4 |
| PCCE (if applicable) | /3 | >= 2 |
| Retention (if applicable) | /5 | >= 4 |
| CTA | /4 | >= 3 |
| Read-Aloud | /3 | >= 2 |

**Verdict:** PASS (all thresholds met) / NEEDS WORK (list items to fix)

      Yeah, it rarely does long-term. Here's the structural problem with Scrum:

      Two-week sprints from a backlog means you're always asking

      Then you add more process to improve the estimates. And so on.

      Instead of 'how long will this take?' ask 'how much is it worth?'

      If it doesn't fit in six weeks, you either narrow the scope

      The default should be: this does not require a meeting.
      Write a document. Post it. People respond when they're ready.
